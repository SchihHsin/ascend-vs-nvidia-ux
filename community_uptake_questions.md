# 社区承接 · 问题清单

> 用途：为「社区承接」tab 提供问题清单。**打分规则由计算方自行定义，本文件只给问题，不含任何评分口径。**
> 问题源自姊妹研究 opknow 的人机协同时序图（算子开发场景）。

---

## 背景：时序图为什么会转向社区

一次算子开发沿时间串起 5 次搜索 **D → M → X → O → L**；在三类节点，工作流会转向社区：

1. **AI 检索官方失败**（官方 how-to 页 SPA 受阻）→ 转向二手社区 / 人工核查官方
2. **人工不确定、独立复核**（安全网）→ 官方 + 二手
3. **需要决策**（版本 / soc / 精度）→ 官方文档 + Gitee 样例 + 论坛

## 用户带去社区的问题（昇腾侧 · 算子开发场景）

| 时序图节点 | 转向社区的触发 | 用户带去社区的问题 | 转向的目的地 |
|---|---|---|---|
| **D · 写 kernel** | 官方 how-to 页 SPA 受阻 | 用 Ascend C 写一个自定义算子的完整流程是什么？msOpGen 怎么生成工程、add_custom.json 怎么配置、aclnn 怎么注册并调用？ | 二手社区 / 官方文档 |
| **M · Tiling** | 需要切分策略、官方说明不足 | Ascend C 算子的 Tiling 怎么切？TilingData 结构如何定义、如何与核函数对接、多核如何分配？ | 官方文档 / 二手社区 |
| **X · 内存越界** | 编译 / 运行报错、定位困难 | Ascend C 算子内存越界 / 地址错误如何定位？是否有 msSanitizer 一类的工具，怎么用？ | 官方文档 / 论坛 / 二手社区 |
| **O · 注册集成** | 接入框架 | 自定义算子如何注册进 PyTorch？torch_npu 调用自定义 aclnn 算子的写法是什么？ | 官方文档 / Gitee 样例 / 二手社区 |
| **L · 精度排查** | 精度与标杆对不齐 | 自定义算子精度与 GPU / CPU 标杆对不齐，如何逐层比对、如何排查？ | 二手社区 / 论坛 |
| **决策 · soc/版本** | 需填 soc_version、配版本 | 我的机器是 Atlas 某型号，soc_version 填什么？当前 CANN、torch_npu、PyTorch 三者版本如何配套？ | 官方文档（版本配套表）/ Gitee 样例 / 论坛 |
